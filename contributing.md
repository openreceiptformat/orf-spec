# Contributing to Open Receipt Format

Thank you for your interest in contributing! This document explains how to participate.

## Ways to Contribute

### 1. Provide Feedback on the Spec
- Read the [specification](spec/ORF-v1.0.md)
- Open an [issue](https://github.com/openreceiptformat/orf-spec/issues) if you find:
  - Unclear language
  - Missing use cases
  - Technical problems
  - Implementation difficulties

### 2. Suggest Improvements
- Start a [discussion](https://github.com/openreceiptformat/orf-spec/discussions)
- Propose new optional fields
- Share real-world edge cases
- Suggest better examples

### 3. Contribute Examples
- Add example receipts showing different scenarios
- Include receipts from your country/region
- Document special cases (returns, exchanges, etc.)

### 4. Build Implementations
- Create libraries in your favorite language
- Build tools (validators, converters)
- Share POS integrations
- Build consumer apps

### 5. Improve Documentation
- Fix typos and grammar
- Clarify confusing sections
- Add diagrams or illustrations
- Translate to other languages

## Contribution Process

### For Small Changes (typos, examples)
1. Fork the repository
2. Make your changes
3. Submit a pull request
4. We'll review and merge quickly

### For Spec Changes (new fields, breaking changes)
1. Open a [discussion](https://github.com/openreceiptformat/orf-spec/discussions) first
2. Get feedback from the community
3. Once there's consensus, create a pull request
4. Working group will review and vote

## Code of Conduct

We are committed to providing a welcoming and inclusive environment. Please:
- Be respectful and constructive
- Welcome newcomers
- Focus on what's best for the standard
- Accept feedback gracefully

Unacceptable behavior includes harassment, trolling, or discrimination of any kind.

## Questions?

- Ask in [Discussions](https://github.com/openreceiptformat/orf-spec/discussions)
- Email: hello@openreceiptformat.org (coming soon)
